Walkthrough

In this documentation we are going to learn how to create walkthroughs. The first thing we should know is that a walkthrough have the following params:

  • name (string)
  • slug (string)
  • steps (array)

The most basic thing before we start a new walkthrough is to know the difference between actions and events.

Actions: They are steps handled and triggered by us (the creator of the walkthrough). For example, at step 3 we we decided to change the active layers of the map, the range of dates...

Events: They are steps where we expect an iteraction from the user. For example, we expect a change on the timeline, a change in the layers...

Steps

First of all, each step must have a type, it belongs to a category (it can only have one) and it has an 'options' param. Those are the basic params that a step should have.

  • type - (string)
  • category - (string)
  • options (object)

Types

The difference between action and event is that an action will be triggered by us and an event will be triggered by the user.

  • Action steps - action
  • Event steps - event

Action Steps

Categories

  • Layers - layers
  • Tab - tab
  • Timeline - timeline
  • Country - country
  • Analysis - analysis
  • Map - map

Layers

We need to set a layers param equal to an array of layer slugs. You can find the slugs here Layerspec. If you want to set a country layer, you should first set the country

Tab

We can set an active tab or a subtab (like the analysis ones).

  • Country => 'countries-tab'
  • Analysis => 'analysis-tab'
  • Basemaps => 'basemaps-tab'
  • High res => 'hd-tab'
{% include code/tab.html %}

Timeline

We need to set a 'dates' param equal to an object with a 'begin' and 'end' param. Of course, the dates should be between the range of dates allowed by the current forest change layer. The format of the dates should be 'dd/mm/YYYY'

{% include code/timeline.html %}

Country

We need to set an 'iso' param with 2 params: 'country' and 'region'

{% include code/country.html %}

Analysis

We must specify the type of the analysis, so the 'analysis' param is mandatory. we have 3 different types of analysis:

  • 'draw' - Draw or upload shape
  • 'country' - Country or region
  • 'area' - Area
{% include code/analysis.html %}

Map

We have different possibilities here. We can set a zoom level, a location or a basemap. Just remember that you can only change one thing per step. For the basemaps, here you have a list of the ids of the basemaps.

  • Default => 'grayscale'
  • Terrain => 'terrain'
  • Satellite => 'satellite'
  • Dark Matter => 'dark'
  • Positron => 'positron'
  • Roads => 'hybrid'
  • Tree height => 'treeheight'
